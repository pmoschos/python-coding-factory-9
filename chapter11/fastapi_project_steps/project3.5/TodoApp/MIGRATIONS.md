# Οδηγός Alembic Migrations (TodoApp)

Αυτός ο οδηγός εξηγεί αναλυτικά τη διαδικασία εγκατάστασης και χρήσης του **Alembic** για τη διαχείριση της Βάσης Δεδομένων σας, χωρίς να χάσετε τα υπάρχοντα δεδομένα σας.

---

## 1. Αρχικοποίηση (Initialization)

Όταν ξεκινάτε με το Alembic για πρώτη φορά στο Project σας, πρέπει να τρέξετε (μέσα στον φάκελο `TodoApp`):

```bash
alembic init alembic
```

*Αυτό δημιουργεί τον φάκελο `alembic/` (με το αρχείο `env.py`) και το βασικό αρχείο ρυθμίσεων `alembic.ini`.*

---

## 2. Παραμετροποίηση Αρχείων (Μόνο 1 φορά)

Προτού τρέξετε οποιοδήποτε migration, πρέπει να συνδέσετε το Alembic με τον κώδικα του FastAPI και τη βάση `todosapp.db`.

### 2.1 Επεξεργασία του `alembic.ini`
Βρείτε τη μεταβλητή `sqlalchemy.url` (περίπου στη γραμμή 85) και αλλάξτε την ώστε να δείχνει στο τοπικό σας SQLite αρχείο:

```ini
# Αρχική μορφή:
# sqlalchemy.url = driver://user:pass@localhost/dbname

# Τελική μορφή (όπως το γράψαμε εμείς):
sqlalchemy.url = sqlite:///todosapp.db
```

### 2.2 Επεξεργασία του `alembic/env.py`
Η βάση πρέπει να μελετήσει τα δομικά χαρακτηριστικά των αρχείων. Ανοίξτε το αρχείο `env.py` και προσθέστε τα models σας.
Βρείτε το πεδίο `target_metadata` (περίπου στη γραμμή 20).

```python
# Αρχική μορφή:
# target_metadata = None

# Τελική μορφή:
import models
target_metadata = models.Base.metadata
```
*Με αυτό τον τρόπο, το Alembic αποκτά πρόσβαση στο επιθυμητό Schema (π.χ. τι columns βάλατε στο Class).*

---

## 3. Διαδικασία Migration (Όταν αλλάζετε κάτι στα Models)

Ας πούμε ότι μόλις προσθέσατε ένα `phone_number = Column(String)` στο `Users` model. 
Πρέπει να περάσετε αυτήν την αλλαγή στη βάση. Σε αυτήν τη φάση τρέχετε τα εξής **ΔΥΟ βήματα με σειρά**:

### ΒΗΜΑ 3Α: Ανίχνευση και Δημιουργία Migration (--autogenerate)
```bash
alembic revision --autogenerate -m "Add phone number to User"
```
**Τι κάνει:** Διαβάζει το `models.py` και το συγκρίνει με τη δομή του πίνακα στο `todosapp.db`. Βλέπει ότι λείπει ένα column, και δημιουργεί ένα νέο αρχείο (Script) μέσα στον φάκελο `alembic/versions`.

### ΒΗΜΑ 3Β: Έλεγχος Ασφαλείας (Μην το παραλείπετε!)
Πηγαίνετε στον φάκελο **`alembic/versions`**. Ανοίξτε το νεότερο αρχείο που μόλις φτιάχτηκε.
Επιβεβαιώστε ότι μέσα στη συνάρτηση `def upgrade():` κάνει μόνο την αλλαγή που περιμένατε.

> [!CAUTION]
> Αν δείτε να γράφει `op.drop_table` (δηλαδή προσπαθεί να διαγράψει όλους τους πίνακές σας) ή `op.create_table` για πίνακες που υπάρχουν ήδη, **σταματήστε**! Σημαίνει ότι το Alembic δεν κατάφερε να διαβάσει τη ΒΔ σας και βλέπει τη βάση ως εντελώς άδεια.

**Παράδειγμα Σωστού Αρχείου:**
```python
def upgrade():
    # Αυτό είναι το σωστό. Απλά προσθέτει μία στήλη. Δεν διαγράφει τίποτα!
    op.add_column('users', sa.Column('phone_number', sa.String(), nullable=True))
```

### ΒΗΜΑ 3Γ: Εφαρμογή της Αλλαγής (Upgrade)
Αν είστε βέβαιοι ότι το αρχείο του Migration στο `versions` είναι σωστό, τρέξτε στο τερματικό:

```bash
alembic upgrade head
```
**Τι κάνει:** Τρέχει τη συνάρτηση `upgrade()` του migration στην πραγματική βάση δεδομένων `todosapp.db`.

Τώρα το `phone_number` μπήκε στον πίνακα των Users και τα παλιά σας δεδομένα δεν χάθηκαν! 🎉
